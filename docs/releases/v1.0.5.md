# Release v1.0.5

**versionName:** `1.0.5` · **versionCode:** `10005`

## Summary

Bug-fix release for the home-screen **Monitoring** toggle, plus a small test-infrastructure fix. Also includes **temporary shortened polling timers** for testing (see warning below).

## Changes

### Monitoring toggle — confirm before flipping (bug fix)
- **Fixed:** rejecting the "turn off monitoring" confirmation left the switch showing OFF while monitoring actually stayed ON (UI/state mismatch).
- Reworked the switch to a **command + single-source-of-truth** model: a tap is treated purely as user intent via `setOnClickListener`, and the switch's displayed state is driven solely by the `monitoringEnabled` flow.
  - Turning **off** no longer flips the switch immediately — it stays ON and shows the confirmation dialog; it only turns off after **Yes**. **No** / dismiss is a true no-op (setting, service, and UI all unchanged).
  - Because intent comes from `onClick` (not `onCheckedChange`), programmatic `isChecked = …` and **view-state restoration** never re-trigger the enable path — this removes a latent bug where rotation/recreation while monitoring was ON re-ran `StreamMonitorService.start(showWelcome = true)` and re-posted the welcome notification.

### Tests
- Fixed `MainFragmentTest.historyListRendersSeededEvents()` which had a non-`Unit` expression body (JUnit "method should be void"), an error that had been blocking the **entire** test class from loading.

### ⚠️ Temporary testing timers (revert before shipping)
- Check interval **2 min → 5s** (`StreamMonitorService.CHECK_INTERVAL_MILLIS`, marked `TODO: TEMP`).
- Retry back-off **10/20/40s → 2/4/6s** (`StreamRepository.RETRY_BACKOFF_MILLIS`, marked `TODO: TEMP`).

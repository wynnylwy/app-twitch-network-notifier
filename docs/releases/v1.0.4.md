# Release v1.0.4

**versionName:** `1.0.4` · **versionCode:** `10004`

## Summary

This branch bundles UI/UX refinements with a reworked stream-monitoring alert system. It builds on the portrait-lock and settings work and adds reliable offline/back-online alerting, plus notification taps that open the app.

## Changes

### Monitoring & alerting (offline-detection)
- Poll interval shortened from **5 min → 2 min**.
- Streamer-offline is **confirmed with 3 retries** (10s / 20s / 40s backoff) before alerting — but only on a healthy→problem transition. Once already in a problem state, each cycle is a single quick check (no repeated 70s of retries).
- Repeat reminders **capped at 3 per problem streak**, then quiet until the status changes. `OFFLINE` and `CONNECTION_ISSUE` are tracked separately.
- Repeat problem reminders are **prefixed with their attempt number** (e.g. `#2: Your live stream is offline, please check.`), and each alert uses a **unique notification id** so reminders **stack** in the shade instead of overwriting one another. The counter resets to `#1` after recovery.
- A single **"back online"** alert on recovery to LIVE.
- History records only real status transitions (repeat reminders aren't logged).
- Alert collector is guaranteed subscribed before the first check emits, so the first alert of a session can't be dropped (replay-0 `SharedFlow` race).

### Notifications
- **Tapping a notification now opens the app.** Both the monitoring and alert notifications carry a launcher `PendingIntent`, so a tap resumes the existing app (without recreating the screen) or cold-starts it if it was killed — matching the behaviour of tapping the home-screen icon (`getLaunchIntentForPackage`).

### UI & app refinements (earlier work on this branch)
- Locked the app to **portrait orientation**.
- Refined the **monitoring toggle**, settings navigation, and history display.
- Centered the settings **save button** and surfaced save errors.
- Distinct application ID for debug builds; version bumps (1.0.1 → **1.0.4**).
